-----------------WORKING WITH DYANMIC PATH IMPORTS ON VSCODE AND TSCONFIG----------

1) In your tsconfig.json file, overwrite the props baseUrl appointing to src and 
the paths property appoint for the most used folders in your app to avoid passing 
slash and dots (../../) to do importations. Example:

     "baseUrl": "./src",                                  
     "paths": {
        "@modules/*" : ["modules/*"],
        "@config/*" : ["config/*"],
        "@shared/*" : ["shared/*"],
        "@errors/*" : ["errors/*"],
     },  

-----------------

2) In any place of your application you can import same file from these folder just 
adding the "@folder_name". Example:

import { AppError } from "@errors/AppError"

Obs, if you're working with typeorm/docker and get some error, add the lib tscon-
ifg-paths running yarn add tsconfig-paths -D  and add the flag 
-r tsconfig-paths/register to your server and typeorm scripts. Example:

"dev": "ts-node-dev -r tsconfig-paths/register --transpile-only --respawn src/server.ts",
"typeorm": "ts-node-dev -r tsconfig-paths/register ./node_modules/typeorm/cli",
